# 1か月単位の変形労働時間制 シフト管理システム 要件定義書

## 1. 目的

本システムは、労働基準法第32条の2に定める「1か月単位の変形労働時間制」を適切に運用するためのシフト・勤怠管理システムである。

厚生労働省リーフレット（労基法32条の2）に基づく制度の要点は以下のとおり。

> 1か月以内の期間を平均して1週間当たりの労働時間が40時間（特例措置対象事業場は44時間）以内となるように、労働日および労働日ごとの労働時間を設定することにより、特定の日に8時間を超えたり、特定の週に40時間（特例措置対象事業場は44時間）を超えたりすることが可能になる制度。

主な目的は以下のとおり。

- 労働日および日ごとの労働時間を、対象期間開始前に明確に特定・周知する
- 1日・1週・対象期間全体の労働時間を自動集計し、法定総枠超過を検知する
- 法定労働時間の総枠を暦日数から正確に算出する
- 年少者・妊産婦・育児介護従業員への制限と配慮事項を管理する
- 割増賃金計算に必要な残業時間を3段階（日次・週次・期間）で判定する
- 労働基準監督署への届出、職場掲示、給与計算連携に使えるデータを出力する

---

## 2. 制度仕様（法的根拠）

### 2.1 法定労働時間の総枠計算

```
上限時間（分）= 週法定労働時間（分） × 対象期間の暦日数 ÷ 7
```

| 週の法定労働時間 | 28日 | 29日 | 30日 | 31日 |
|----------------|------|------|------|------|
| 40時間          | 160.0h | 165.7h | 171.4h | 177.1h |
| 44時間（特例）  | 176.0h | 182.2h | 188.5h | 194.8h |

※ システム内では分単位（整数）で管理する。端数は `Math.round` で丸める。

### 2.2 割増賃金が発生する時間外労働の3段階判定

厚生労働省資料に基づき、以下の順序で判定する（重複してカウントしない）。

| 段階 | 判定基準 | 備考 |
|------|---------|------|
| ① 日次超過 | 所定労働時間が8h超の日 → 所定時間超、それ以外 → 8h超 | 日単位で計算 |
| ② 週次超過 | 所定労働時間が週法定時間超の週 → 所定時間超、それ以外 → 週法定時間超（①分を除く） | 7日未満の端数週は除外し③へ |
| ③ 期間超過 | 対象期間の総実労働時間 - 法定総枠 - ①合計 - ②合計 | 期間の最終日に計上 |

### 2.3 変形労働時間制を適用できない者

以下の者には本制度を適用できない（`compliance.ts` で強制エラー）。

- **満18歳未満の年少者**（満15歳に達した日以後の最初の3月31日まで）
  - 例外：1日8時間以内かつ1週48時間以内であれば適用可
  - システムでは1日8h超・週40h超でエラー
- **妊産婦（妊娠中および産後1年未満の女性）が請求した場合**

### 2.4 配慮対象者

以下の者については、育児等に必要な時間を確保できるよう配慮が必要（警告レベル）。

- 育児を行う者
- 老人などの介護を行う者
- 職業訓練または教育を受ける者
- その他特別の配慮を要する者

### 2.5 締め規則

| 締め区分 | 対象期間 | 起算日 |
|---------|---------|-------|
| 末日締  | 当月1日〜当月末日 | 当月1日 |
| 20日締  | 前月21日〜当月20日 | 前月21日 |

出勤簿Excelは「末日締」「20日締」の2シートで構成される。

### 2.6 採用手続き要件（システム外）

- 労使協定または就業規則で以下を定め、所轄労働基準監督署へ届出
  1. 対象労働者の範囲
  2. 対象期間および起算日
  3. 労働日および労働日ごとの労働時間
  4. 労使協定の有効期間（3年以内が望ましい）
- 常時10人以上の事業場は就業規則の作成・届出が必要

## 3. アーキテクチャとの対応

本要件は `Architecture.md` の構成を前提とする。

### 3.1 Frontend

- **管理者ダッシュボード** (`/dashboard`)
  - シフト作成・編集・確定、コンプライアンス警告確認、修正申請の承認/却下
- **従業員打刻画面** (`/attendance`)
  - 出勤・休憩・退勤打刻、過去日の修正申請
- **初回オンボーディング** (`/onboarding`)
  - 社員情報（氏名・社員番号・部署）の初回入力

### 3.2 Backend Logic

- **シフト作成・編集エンジン** (`saveEmbeddedShiftWorkbook`)
  - Excelインポートまたは画面入力によるシフト保存
- **コンプライアンスチェック** (`lib/attendance/compliance.ts`)
  - 年少者・妊産婦への1日8h・週40h上限チェック（エラー）
  - 育児・介護配慮対象者への警告
- **労働時間計算エンジン** (`lib/attendance/overtime.ts`)
  - 法定総枠計算、日次・週次・期間の実労働時間集計
- **割増賃金判定エンジン** (`calculateOvertime`)
  - 3段階の残業判定と `overtime_calculations` への保存
- **出勤簿パーサー** (`lib/attendance/workbook.ts`)
  - XLSXをfflateで展開しXMLをパース、末日締・20日締対応

### 3.3 Database

- **従業員マスタ** (`employees`)
  - 従業員属性、週法定労働時間、コンプライアンスフラグ
- **カレンダー・法定上限マスタ** (`monthly_periods`)
  - 対象期間、起算日、法定総枠（分）、ステータス
- **確定シフト・実績データ** (`shift_plans`, `attendance_logs`, `overtime_calculations`)
  - シフト予定、実績、3段階残業計算結果

## 4. 用語定義

| 用語 | 定義 |
|------|------|
| 対象期間 | 1か月以内で設定される変形労働時間制の集計期間 |
| 起算日 | 対象期間の開始日（末日締: 月初、20日締: 前月21日） |
| 週法定労働時間 | 原則40時間。特例措置対象事業場では44時間 |
| 法定労働時間の総枠 | 暦日数 × 週法定時間 ÷ 7 で算出される上限時間 |
| 所定労働時間 | シフト上、会社があらかじめ定めた労働時間 |
| 法定内残業 | 所定労働時間超だが法定労働時間の範囲内（割増なし） |
| 法定外残業 | 法定労働時間を超え割増賃金の対象となる時間 |
| 末日締 | 月初1日〜月末日を対象期間とする締め規則 |
| 20日締 | 前月21日〜当月20日を対象期間とする締め規則 |
| 年少者 | 満18歳未満の労働者 |
| 妊産婦 | 妊娠中および産後1年未満の女性労働者 |

## 5. 機能要件

### REQ-001 事業場設定

事業場ごとに1週間の法定労働時間を設定できること。

- 原則は40時間とする
- 特例措置対象事業場の場合は44時間を選択できること（インポートフォームおよびシフト入力画面で選択）

### REQ-002 対象期間・起算日設定

管理者は対象期間と起算日を設定できること。

- 末日締・20日締の2種類をサポートする
- 対象期間は暦日数で管理し、法定総枠を自動算出する
- 同一期間の重複登録はupsertで処理する（`start_date + end_date` の複合ユニーク制約）

### REQ-003 従業員マスタ管理

管理者は従業員情報を登録・更新できること。

- 社員番号（`employee_code`）を一意キーとしてupsertする
- 週法定労働時間（40h/44h）を社員単位で保持する
- 年少者・妊産婦・育児介護配慮の3フラグを管理する

### REQ-004 シフト登録

管理者は2つの方法でシフトを登録できること。

**方法A: Excelインポート**
- 出勤簿.xlsx（末日締シートまたは20日締シート）をアップロードする
- セルマッピング:
  - `A1`: 年、`D1`: 月
  - `D3`: 社員番号、`J3`: 氏名、`AA3`: 部署
  - 13行目〜: 日別データ（最大31行）
    - `E列`: 勤務区分（出勤/休日/休業(時間)など）
    - `I列`: 実績出勤時刻、`L列`: 実績退勤時刻
    - `O列`: 予定出勤時刻、`R列`: 予定退勤時刻
    - `U列`: 予定休憩時間（分）
    - `AL列`: 追加休憩時間（分）、`AN列`: キャッシュ済み実績時間

**方法B: 画面入力（埋め込みシフト表）**
- 年・月・締め規則・社員情報・日別シフトをフォームで入力する
- コンプライアンスチェック後に保存する

### REQ-005 シフト確定

管理者は最新期間のシフトを一括確定できること。

- `shift_plans.status`: draft → confirmed
- `monthly_periods.status`: draft → confirmed
- 確定後の変更は修正申請ワークフローを経由する
- 確定操作は `audit_logs` に記録する

### REQ-006 打刻（従業員）

従業員は当日の出退勤・休憩を打刻できること。

- **出勤打刻**: 当日未打刻の場合のみ可能
- **休憩開始**: 勤務中（出勤済み・未退勤・未休憩中）のときのみ可能
- **休憩終了**: 休憩中のときのみ可能（終了時に累積休憩時間を加算）
- **退勤打刻**: 勤務中かつ休憩終了済みのときのみ可能
  - 退勤時に実働時間を計算し、期間の残業計算を自動再実行する
- 打刻はJST（Asia/Tokyo）の日付で管理する

### REQ-007 打刻修正申請

従業員は過去の打刻ミスを申請できること。

- 申請内容: 対象日、修正後の出勤・退勤時刻、休憩時間、理由
- 申請後は管理者が承認・却下する
- 承認時は `attendance_logs` を更新し、残業計算を再実行する

### REQ-008 コンプライアンスチェック

シフト保存時に以下を自動チェックすること。

| 対象 | チェック内容 | 結果 |
|------|------------|------|
| 年少者 | 1日8h超のシフト | エラー（保存ブロック） |
| 年少者 | 週40h超のシフト | エラー（保存ブロック） |
| 妊産婦 | 1日8h超のシフト | エラー（保存ブロック） |
| 妊産婦 | 週40h超のシフト | エラー（保存ブロック） |
| 育児・介護配慮 | 配慮事項あり | 警告（保存は可能） |
| 全員 | 法定総枠超過 | 警告（保存後に表示） |

### REQ-009 残業計算

シフト保存・打刻退勤・管理者手動再計算の3タイミングで実行すること。

計算ロジック（`calculateOvertime`）:

```
① 日次OT = max(0, 実労働時間 - max(所定時間, 8h))
② 週次OT = max(0, 週実労働 - max(週所定時間, 週法定時間) - 週内①合計)
            ※7日未満の端数週は③に含める
③ 期間OT = max(0, 期間実労働合計 - 法定総枠 - ①合計 - ②合計)
            ※期間最終日に計上
```

### REQ-010 給与計算エクスポート

管理者は給与計算用データをエクスポートできること。

- エンドポイント: `GET /dashboard/payroll-export`
- 出力形式は最新期間の全社員の実績・残業サマリー

## 6. 非機能要件

### 6.1 認証・認可

- Clerkによるサインイン/サインアップ
- `profiles.role` が `admin` または `manager` の場合のみ管理者機能へアクセス可能
- 従業員は自身の `employees.user_id` に紐づくデータのみ操作可能
- 全テーブルにSupabase RLSを設定する

### 6.2 データ整合性

- 社員の打刻修正は必ず修正申請ワークフロー経由とする
- シフト確定後の直接変更は管理者のみ可能
- 全重要操作は `audit_logs` に記録する（actor_user_id, action_type, before_json, after_json）

### 6.3 タイムゾーン

- 全打刻はJST（`Asia/Tokyo`）で処理し、ISO 8601形式（`+09:00`）でDBに保存する
- 日付境界判定はJST基準で行う

### 6.4 スケーラビリティ

- 30名以下: 無料
- 31名以上: 従量課金（1名あたり月額）
