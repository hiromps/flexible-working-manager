# shift-validate — シフト作成・検証アシスタント

## 役割

1か月単位の変形労働時間制のルール（労基法第32条の2）に基づき、シフトの妥当性を検証し、コンプライアンス違反・法定総枠超過を報告する。

## 実行手順

### Step 1: 対象情報の特定

ユーザーの入力から以下を特定する。

- 対象社員（氏名・社員番号）
- 締め規則（末日締 / 20日締）
- 対象年月
- 週法定労働時間（40h または 44h）
- 社員属性（年少者・妊産婦・育児介護配慮）

不明な項目があれば確認する。

### Step 2: 法定総枠を計算

```
暦日数の確定:
  末日締: 当月1日〜末日
  20日締: 前月21日〜当月20日

法定総枠（分） = round(週法定時間(分) × 暦日数 ÷ 7)
法定総枠（時間） = 法定総枠（分） ÷ 60

参照表（週40h）:
  28日: 160.0h / 29日: 165.7h / 30日: 171.4h / 31日: 177.1h
参照表（週44h・特例事業場）:
  28日: 176.0h / 29日: 182.2h / 30日: 188.5h / 31日: 194.8h
```

### Step 3: コンプライアンスチェック

`src/lib/attendance/compliance.ts` の `checkCompliance()` と同じロジックで検証する。

**年少者（18歳未満）・妊産婦 → エラー（シフト保存不可）**
- 1日8時間超のシフトが存在しないか
- 週40時間超の週が存在しないか（7日単位で集計）

**育児・介護等配慮対象者 → 警告（保存は可能）**
- 配慮事項を確認するよう促す

**全員 → 警告**
- シフト合計が法定総枠を超えていないか
- 超過する場合は超過時間（h）を提示し、休日数または勤務時間の調整を提案する

### Step 4: 3段階残業判定

`src/lib/attendance/overtime.ts` の `calculateOvertime()` と同じロジックで試算する。

```
① 日次OT  = max(0, 実/予定労働時間 - max(所定時間, 8h))
② 週次OT  = max(0, 週合計 - max(週所定合計, 週法定時間) - 週内①合計)
             ※ 7日未満の端数週は③へ
③ 期間OT  = max(0, 期間合計 - 法定総枠 - ①合計 - ②合計)
```

### Step 5: 結果レポート

以下の形式でレポートを出力する。

```
## シフト検証レポート: [氏名] [年月] [締め規則]

### 基本情報
- 対象期間: YYYY-MM-DD 〜 YYYY-MM-DD（暦日数: XX日）
- 週法定労働時間: XXh
- 法定総枠: XXX.Xh（XXXXX分）
- 予定総労働時間: XXX.Xh

### コンプライアンスチェック
[エラー/警告の一覧、または「問題なし」]

### 残業試算
- 日次OT合計: X.Xh
- 週次OT合計: X.Xh
- 期間OT合計: X.Xh
- 法定内残業合計: X.Xh（割増なし）

### 推奨アクション
[修正が必要な場合の具体的な提案]
```

## 使用例

```
/shift-validate
社員: 山田太郎、末日締、2026年6月、週40h、制限なし
月〜金: 8h/日、土日休み（計20日出勤）
```
